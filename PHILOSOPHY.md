# AEGIS-Ω: Fractal UI Philosophy

## Overview
The Fractal UI is a design framework built for AGI orchestration. It rejects the "flat" nature of traditional web apps in favor of a recursive, multi-scale environment that mirrors the complexity of the underlying intelligence.

## Core Principles

### 1. Recursive Information Density
*   **Definition**: Every component is a fractal of the whole.
*   **Application**: A "Node" component contains a mini-dashboard. A "Log" entry contains a mini-graph. There is no "bottom" to the data; only deeper layers of resolution.

### 2. Contextual Semantic Zoom
*   **Definition**: Navigation is a change in scale, not a change in location.
*   **Application**: Transitions use `scale` and `opacity` to create the feeling of "diving" into a sub-system. Information is revealed only when the user is at the appropriate "altitude."

### 3. Self-Similarity Across Scales
*   **Definition**: The same visual rules apply at every level.
*   **Application**: The 8x8 grid of the `SigmaMatrix` is mirrored in the background lattice and the layout of the `NodeManager`. This creates a "Universal Substrate" feel.

### 4. Emergent Manifold Navigation
*   **Definition**: The UI is a dynamic space shaped by system state.
*   **Application**: Navigation paths are highlighted based on AGI "Attention" (saliency). The UI guides the user to where the intelligence is currently most active.

## AGI Aspect Mapping

### Learning (Synaptic Growth)
*   **Visual**: Branching fractal structures in the `Epinoetic Foundry`.
*   **Interaction**: Drag-and-drop "Pruning" of low-confidence weights.

### Reasoning (Inference Paths)
*   **Visual**: Glowing "Logic Trees" in `SwarmChat`.
*   **Interaction**: Clicking a node in the tree reveals the specific prompt/response pair for that sub-reasoning step.

### Perception (The Sigma Retina)
*   **Visual**: High-frequency ripples in the `SigmaMatrix`.
*   **Interaction**: Hovering over a cell "magnifies" the raw data stream behind the abstraction.

### Goal-Setting (The Alignment Attractor)
*   **Visual**: A central "Strange Attractor" in the background that pulls UI elements toward it.
*   **Interaction**: Setting a "Directive" shifts the attractor's position, causing the UI to re-orient.

### Self-Awareness (System Introspection)
*   **Visual**: A "Daemon Mode" where the UI displays its own source code and resource metrics in a recursive loop.
*   **Interaction**: Real-time editing of UI parameters through a "Whisper" terminal.
